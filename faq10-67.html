Active Directory（LDAP）連携時にまれに送信エラーになったり、POP受信エラーが発生するとき

アカウントのActive Directory（LDAP）連携をしているとき、E-Post Mail ServerとAD（ドメインコントローラ）やLDAPサーバとの間で通信に時間がかかることが原因となり、ふだんは正常に送ることができる社内メールで、まれに送信エラーになったり、POP受信エラーになることが発生することがあります。

そのような事象が発生する場合、メールサーバとAD（ドメインコントローラ）やLDAPサーバとの間の通信で少し時間がかかることにより、ユーザーのマッチングが完ぺきにできていないおそれがあります。AD（ドメインコントローラ）やLDAPサーバとの間で、通信タイムアウトの設定を見直してください。

ADとの通信については、下記の設定ファイル項目によって、リトライ回数を上げて試すことができるようになっています。 "ADRetryTime"のキーはデフォルトで作成されていませんので、数値10進でキーを新規作成し、値を10より上の任意の数、たとえば20や30などの数値を入れて試してください。変更後は、該当する各サービスの再起動が必要です。

この"ADRetryTime"ファイルは、epstrd、epstdd、epstimap4d、epstimap4dそれぞれに作成、設定します。


[epstrd] (E-POST SMTP Receiver)
メール作業フォルダ　（既定値 /var/spool/epms/）
→reg
　→epost
　　→ims
　　　→EPST5RS
　　　　→ADRetryTime
　　　　(DWORD Default 10) ADへのユーザ情報問合せリトライ回数

[epstdd] (E-POST SMTP Delivry Agent)
メール作業フォルダ　（既定値 /var/spool/epms/）
→reg
　→epost
　　→ims
　　　→EPST5DS
　　　　→ADRetryTime
　　　　(DWORD Default 10) ADへのユーザ情報問合せリトライ回数

[epstpop3d] (E-POST POP3 Server)
メール作業フォルダ　（既定値 /var/spool/epms/）
→reg
　→epost
　　→ims
　　　→EPST5POP3S
　　　　→ADRetryTime
　　　　(DWORD Default 10) ADへのユーザ情報問合せリトライ回数

[epstimap4d] (E-POST IMAP4rev1 Server)
メール作業フォルダ　（既定値 /var/spool/epms/）
→reg
　→epost
　　→ims
　　　→EPST5IMAP4S
　　　　→ADRetryTime
　　　　(DWORD Default 10) ADへのユーザ情報問合せリトライ回数
