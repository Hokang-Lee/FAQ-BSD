E-Post Mail Server V / E-Post SMTP Server V で使用している設定ファイル〔BSD版〕 
■共有情報−[メール作業ディレクトリ]/regディレクトリ内の設定ファイルで設定
    ※設定ファイルの拡張子種類は、次の通り値の型を表わす。
       （ .0 = BINARY,  .1 = 数値,  .2 = 文字列及びその他の形式 ）
     ※秒数や回数、サイズなどの数値値は原則としてすべて10進数表記で設定。

[メール作業ディレクトリ]　（既定値 /var/spool/epms/）
->reg
 ->epost
   ->ims
     ->UserManager.1          (数値  Default 1) 1:OS Account, 0:独自 Account
     ->Membership.2           (文字列  Default NULL) NULL=OS Accountの場合、マシンのローカルグループを使用
                                       Soft Accountの場合、ユーザアカウントDB名
     ->SendDataTimeout.1    (数値  Default 60)sec (送信タイムアウト時間:default 60秒)
     ->AddressFamily.1        (数値  Default 0) 0:IPv4 only, 1:IPv6 only, 2:IPv4&IPv6 mix
     ->AccountLength.1        (数値  Default 20) Soft Account運用時でのアカウント最大文字数(最大値=64)
     ->FullnameLength.1      (数値  Default 30) Soft Account運用時でのフルネーム最大文字数(最大値=64)
                                        ※FullnameLength.1での指定サイズはAccountLength.1の指定サイズに影響し、
                                        ホーム項の入力サイズにも影響する
     ->HostName.2             (文字列  Default NULL) IPv6 host name.
     ->DomainNamesAreLocal.0  (BINALY Default NULL) 運用ドメインリスト
     ->MailInCopy.0             (BINARY Default NULL) Mail Carbon copy address.
     ->MailInMaxSize.1        (数値  Default 0) 0:無制限 1メール当りの限界サイズ
     ->MailInBoxSize.1        (数値  Default 0) 0:無制限 1メールボックス当りの保存限界サイズ
     ->RCPTMaximum.1       (数値  Default 0) 0:無制限 あて先(RCPT TO:)数の制限
     ->MailInBoxDir.2          (文字列  Default /var/spool/epms/inbox/%USERNAME%) メールボックスディレクトリ
     ->LastMsgId.1             (数値  Default 0) メール受信カウンタ
     ->MailGroup.2              (文字列  Default IMSUsers) OS Accountの場合、メールローカルグループ名
     ->MailSpoolDir.2           (文字列  Default /var/spool/epms/) メール作業ディレクトリ
     ->MailFilterUp.1           (数値  Default 0) 0:しない 1:する メールフィルタ処理の強化
                                      '1'の指定で'*[STRING]*'のようなワイルドカード指定が可能になる
     ->LocalFailuresToPostmaster.1 (数値  Default 0) 0:Disable, 1:Enable エラーメールをpostmasterに送信
     ->PostMaster.2             (文字列  Default administrator) postmasterアカウント
     ->POP3IsNotUsed.1       (数値  Default 0) 0:反映しない 1:反映する  POP3禁止を全アカウントに反映する
     ->SearchLocalTime.1     (数値  Default 1) 0:UTC日付 1:ローカル日付 IMAP4で日付検索を行う場合
                                       従来はデフォルト[UTC日付]、該当バージョンから[ローカル日付]に変更
     ->SearchAccessType.1   (数値  Default 0) 0:更新日時 1:アクセス日時 2:作成日時  IMAP4で日付検索を行う場合
                                       従来はデフォルト[ファイル作成日時]、該当バージョンから[ファイル更新日時]に変更
     ->PasswordLength.1     (数値  Default 14) パスワードの最大入力数を変更 (MAX 64byte,最大値=64)
                                       (EPSTMAN v5.04〜)
     ->SecureAccountDB.1    (数値  Default 0) Soft Accountデータベース内のパスワードの暗号化指定オプション
                                       0:プレーン 1:暗号化
     ->AutoCreateFolder.1     (数値  Default 1) IMAP4ログイン時にルートディレクトリ(POP3の場所と同じ)が存在しない場合
                                       ディレクトリを自動作成する 1:作成する 0:作成しない
     →AutoSubDirAndMsg.1   (数値  Default 1) 削除対象ディレクトリにメールやサブディレクトリが含まれていても
                                       削除可能にする 1:削除する 0:削除しない
     ->aliases                       (ディレクトリ)  エイリアス管理用
       ->00000000               (ディレクトリ)  エイリアスその0
         ->Alias.2                   (文字列  Default NULL) エイリアス名
         ->MappedTo.2            (文字列  Default NULL) 実アドレス
       ->00000001               (ディレクトリ)  エイリアスその1
         ->Alias.2                   (文字列  Default NULL) エイリアス名
         ->MappedTo.2            (文字列  Default NULL) 実アドレス

         ->BouncesTo.2         (文字列  Default NULL) 未送信レポート指定時メールアドレス
         ->CharSet.2           (文字列  Default NULL) UTF-8: ARTICLE INDEXコマンドで返される一覧表示メールのヘッダを

         ->ListPassword.2      (文字列  Default NULL) 投稿パスワード
         ->ListSave.1            (数値  Default 0) 投稿内容を保存 0:保存しない,1:保存する
         ->ListSaveFolder.2    (文字列  Default NULL) 投稿内容保存先
         ->ListSaveExtension.2 (文字列  Default TXT）投稿内容保存ファイルの拡張子の指定 (EPSTRD v5.06〜 EPSTDD v5.05〜)
         ->ListSaveOrign.1     （数値  Default 0) 投稿内容保存ファイルに添付削除の有無を無視して原本メールを保存
                                         0:添付削除の有無に依存 1:原本のまま (EPSTRD v5.06〜)
         ->ListSaveURL.2        (文字列  Default NULL）投稿内容保存ファイルのWebからの参照用URL (EPSTRD v5.06〜)
         ->ListSaveHTMLIndex.1 (数値  Default 1) 投稿内容保存ファイルのHTML形式のインデックスファイルの作成
                                         1:作成する 0:作成しない (EPSTRD v5.06〜)
         ->MlId.1                   (数値  Default 0) カウンタ値
         ->MlToken.2              (文字列  Default NULL) 付加表題＆カウント方法
         ->Moderator.2            (文字列  Default NULL) 管理人アドレス
         ->ModeratorControlJoin.1  (数値  Default 0) 入会要求は管理人へ転送

         ->ReplyTo.1                (数値  Default 1) Reply-To:ヘッダを付加する 0:しない,1:する
         ->WhoCanSend.1        (数値  Default 1) 投稿者の条件 1:メンバーのみ,2:誰でも,3:管理人のみ
         ->Members                (ディレクトリ) 参加者管理用
           ->[mail address]       (ディレクトリ) 参加者メールアドレス
                  : 繰り返し
       ->[ML-list nameN]         (メーリングリスト名１)
         : 以下繰り返し
         
     ->domain
       ->operation                    (ディレクトリ) ドメイン毎の操作パスワード（空白＝パスワード無し）

           ->[domain nameN].2 (文字列) eg.password2
     ->Secur
       ->SysPassword.2            (文字列) メールサーバーの操作パスワード（空白＝パスワード無し）

■個別情報−[メール作業ディレクトリ]/regディレクトリ内の設定ファイルで設定
    ※設定ファイルの拡張子種類は、次の通り値の型を表わす。
       （ .0 = BINARY,  .1 = 数値,  .2 = 文字列及びその他の形式 ）
     ※秒数や回数、サイズなどの数値値は原則としてすべて10進数表記で設定。

[メール作業ディレクトリ]　（既定値 /var/spool/epms/）
->reg
 ->epost
    ->ims
      ->MailSpoolDir.2             (文字列  Default /var/spool/epms/) メール作業ディレクトリ
      ->LISTENADDRPRI.1      (数値  Default 1) リッスンIPアドレスのFQDNをホスト名にする
                                          1:優先する 0:優先しない(ホスト名) (EPSTRD v5.12〜)
      ->domain
        ->acountNumber         (ディレクトリ) ドメイン毎の登録者数上限値 (0＝無制限)
            ->[domain name1].1 (数値  Default 0)
                :
            ->[domain nameN].1
        ->folder                 (ディレクトリ) ドメイン毎のメールボックス保存ディレクトリ位置
            ->[domain name1].2 (文字列) eg.domain1.co.jp
                :
            ->[domain nameN].2 (文字列) eg.domainN.co.jp
        ->smtpip               (ディレクトリ) ドメインに対応するSMTP-IPアドレス（空白＝マシンに設定されている全てのIPに応答）
            ->[domain name1].2 (文字列) eg.192.168.0.3
                :
            ->[domain nameN].2 (文字列) eg.192.168.0.4
        ->pop3ip               (ディレクトリ) ドメインに対応するPOP3-IPアドレス（空白＝マシンに設定されている全てのIPに応答）
            ->[domain name1].2 (文字列) eg.192.168.0.3
                :
            ->[domain nameN].2 (文字列) eg.192.168.0.4
        ->imap4ip              (ディレクトリ) ドメインに対応するIMAP4-IPアドレス（空白＝マシンに設定されている全てのIPに応答）
            ->[domain name1].2 (文字列) eg.192.168.0.3
                :
            ->[domain nameN].2 (文字列) eg.192.168.0.4


[epstrd] (E-POST SMTP Receiver)
    ※設定ファイルの拡張子種類は、次の通り値の型を表わす。
       （ .0 = BINARY,  .1 = 数値,  .2 = 文字列及びその他の形式 ）
     ※秒数や回数、サイズなどの数値値は原則としてすべて10進数表記で設定。

[メール作業ディレクトリ]　（既定値 /var/spool/epms/）
->reg
 ->epost
    ->ims
     ->EPST5RS                 (ディレクトリ)
       ->Timeout.1               (数値  Default 300)sec  無通信状態タイムアウト
       ->SpoolFsSync.1         (数値  Default 1000) incomingディレクトリへの滞留可能最大メール数 0:無制限
       ->MovefileRetry.1       (数値  Default 300000)ms incomingフォルダへの移動リトライ上限(300000ms=30秒)
                                         0:無制限 (v5.02〜)
       ->PatternOfTheOnce.1 (数値  Default 500) フィルタデータパターンの一括処理数。
       ->PasswordFile.2        (文字列  Default apop.dat) SMTP認証パスワード設定ファイル名
       ->ListenMode.1          (数値  Default 0) 0:全てのIP許可, 1:指定IPのみ リッスンIPアドレス制限フラグ
       ->ListenIP.2                (文字列  Default NULL) 指定IPアドレス：ポート設定
       ->ImagePath.2           (文字列  Default NULL) サービスプログラムのインストール先フルパス表記
       ->DefaultCode.2         (文字列  Default us-ascii) us-ascii=ASCII, iso-2022-jp=JIS  自動応答文字コード
       ->RecvDataBufferSize.1(数値  Default 0) 0:システムに依存  socket受信バッファサイズ
       ->AcceptLimit.1          (数値  Default 0) 同時接続数 0:無制限
       ->AcceptlogEnabled.1   (数値  Default 0) 接続マシンログ(acceptlog) 0:Disable, 1:Enable
       ->SMTPAUTHOnly.1      (数値  Default 0) 0:両方 1:認証専用 2:認証ファイル毎 セキュリティレベル
       ->SMTPAUTHMode.2    (文字列  Default NO) PLAIN,LOGIN,CRAM-MD5  SMTP認証方式,  "XOAUTH2"を追加 (v5.14〜)
       ->VirusCheck.1           (数値  Default 0) 0:Disable, 1:Enable メールフィルタ実行フラグ
       ->Received.1              (数値  Default 20) メールループ最大回数
       ->UsedSSL.1              (数値  Default 0) TRUE:SSL使用, FALSE:SSL不使用
       ->Certificate.2            (文字列  Default 空白) SSL/TSL 証明書ファイル名
       ->Private-Key.2           (文字列  Default 空白) SSL/TSL 個人鍵ファイル名
       ->UsedSTARTTLS.1     (数値  Default 1) 0:Disable, 1:Enable  STARTTLS命令を使用
       ->SecureLayOption.1   (数値  Default 3) 1:SSL2.0禁止 2:SSL3.0禁止 3:SSL2.0-3.0禁止(SSL3.0以前禁止)
                                     4:TLS1.0禁止 7:TLS1.0,SSL2.0-3.0禁止(TLS1.0以前禁止) 8:TLS1.1禁止
                                     15:TLS1.1以前禁止 16:TLS1.2禁止 31:TLS1.2以前禁止 (v5.01〜)
                                     ※10進指定。"31"指定でTLS1.2以前禁止となりTLS1.3以降のみ許可
       ->SecureLevel.1          (数値 default:0) SSL_CTX_set_security_levelの設定に相当 レベル:0〜5 (v5.16〜)
       ->SecuerLayCipher.2   (文字列  Default ALL) 利用する暗号アルゴリズムの組み合わせオプション（最大1023文字）(v5.01〜)
                                     ※TLSv1.3の暗号組み合わせ名を":"で区切る。以下が既定値
                                 "TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256:TLS_AES_128_GCM_SHA256"
       ->HidesGreeting.1       (数値  Default 0) グリーティングでの製品名・バージョン表示 0:表示する, 1:非表示
       ->ADRetryTime.1        (数値  Default 10) ADへのユーザ情報問合せリトライ回数
       ->ADRetryMSec.1        (数値  Default 1000)msec  ADへのユーザ情報問合せリトライ間隔時間(単位:ミリ秒)
       ->ADCashLiveTime.1    (数値  Default 0)sec AD連携のアカウント情報問合せ時のキャッシュ機能有効時間,0:無効 
       ->AuthLockOut.1         (数値  Default 0) 認証失敗時に接続元IPからの接続ロックアウト機能とロックアウトまでの回数 0:無効 
       ->AuthLockOutTime.1  (数値  Default 0)min  認証失敗時に接続元IPからの接続ロックアウト期間(単位:分) 
       ->IPLockOut.1             (数値  Default 0) 指定時間内でのIP毎の同時接続可能数 0:制限無し (v5.01〜)
                                     ※下の設定とともに任意値設定で有効に働く。例・10,30（回）"rejectsmtpip.dat" にデータ蓄積
       ->IPLockOutSPTime.1   (数値  Default 0)sec 同上サンプリング間隔(秒単位) (v5.01〜)
                                     ※上の設定とともにサンプリング間隔設定で有効に働く。例・1 (秒)
       ->IPLockOutTime.1      (数値  Default 0)sec 同上ロックアウト時間(秒単位) (v5.01〜)
                                     ※上2つの設定によるロックアウト時間。例・300,60 (秒)
       ->Parameters               (ディレクトリ)
         ->PortNo.1                 (数値  Default 25)
         ->MailInlogEnabled.1  (数値  Default 0) SMTP受信ログ(inlog) 0:Disable, 1:Enable
         ->MailQueueDir.2        (文字列  incoming) メール受信ディレクトリ名

[epstdd] (E-POST SMTP Delivry Agent)
    ※設定ファイルの拡張子種類は、次の通り値の型を表わす。

                           2=SMTP-AUTHを[ID]|[PW]情報をBASE64エンコードされている文字列を使って実施する。

       ->SaveDead.1              (数値  Default 1) 0:しない 1:する deadディレクトリに保管するか否か
       ->LogicalIP.1                (文字列  Default NULL) 空白(NULL)は常に配送実行、IPアドレスを設定すると該当IP獲得時のみ配送実行
