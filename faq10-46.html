「管理者にもエラーメールを送信する」チェックボックスについて

E-POSTコントロールセンターのメールサーバ管理から［システム管理メニュー］をクリック、［基本設定］を開いて「管理者アカウント」項目を確認すると、「管理者にもエラーメールを送信する」チェックボックス項目があります。デフォルトではチェックボックスがオフになっています。
オンにしたときと、オフのままのときの違いは、下記のようになります。「管理者にもエラーメールを送信する」部分の「エラー」とは、宛先不明や User Unknown. などの相手先サーバの拒絶を含めたメール不達に関するものであり、サービスプログラムなどのシステム的な問題などについて警告を発する類のものではないことを理解してください。

まず、チェックボックスをオンにした場合は、宛先不明や User Unknown. などのリターンがあったときに E-Post Mail Server システム内で管理されている postmaster アカウントより、エラーメールが自動生成され、指定された管理者と送信者の両方に届けられます。送信者以外に管理者も、宛先不明や User Unknown. などを知ることができます。

次に、チェックボックスがオフのままの場合は、宛先不明や User Unknown. などのリターンがあったときに E-Post Mail Server システム内で管理されている postmaster アカウントより、エラーメールが自動生成され、送信者自身に届けられます。送信者が、宛先不明や User Unknown. などを知ることができます。

運用する際、チェックボックスをオンにするか、オフにするかは、状況とポリシーに依存します。メール不達をすべて把握しておこうというならチェックボックスはオンで運用するとよいでしょうし、常にすべてを把握するほどではないという判断でしたらチェックボックスはオフでよいと思います。このことは管理者の負担にもかかわります。

いずれも、メール不達の原因を正しく探ろうとした場合、配送失敗ログ（faillog）、配送時の詳細ログ（senderlog）などを取得しておき、ログから精査する必要があることは変わりありません。

チェックボックスをオンにした場合、管理者アカウントでしっかりメールを受信し読むことが大前提です。当然ですが、管理者アカウント宛に来たメールを一切読まないで放置することは避けてください。思わぬ量のバウンスメールが来る可能性があります。
